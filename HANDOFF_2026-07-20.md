  spread / entropy / mean reward, or set epochs>1 and print the later-epoch loss.

## 3) COMPLETE LIST OF ISSUES

### A) RESOLVED this session — durability of the "keep only if better" ratchet
A 4-agent review team (correctness, durability, two-inputs, speed-to-consistency) reviewed the heart
TWICE. Fixed and verified:
1. **Pseudo-replication (CRITICAL, 3 reviewers found it).** The gate counted 512 episodes as
   independent, but they came from ~14–23 distinct days and greedy determinism made ~22 episodes/day
   identical → one lucky day faked overwhelming significance → ratchet drifts up on noise. **FIX:**
   `_day_paired` counts DISTINCT DAYS, not episodes. Unit test proves the exact case now rejects.
   Monte-Carlo: false-adopt back to ~1.1%/gen (was inflated).
2. **Winner's curse / multiple comparisons.** Screening K candidates and adopting the best inflates
   false-adopts. **FIX:** screen → single best → **two independent fresh confirms** vs the frozen
   champion (squares per-gen false-adopt to ~1e-4 → O(1) over 5000 gens).
3. **Long-run non-backslide.** No rollback/anchor. **FIX:** audit HIGH-WATER anchor at a fixed
   reference seed; an adopt is blocked if the frozen anchor paired-beats it out of sample; anchor rises
   only on a paired OOS win (no self-inflation). `meta_anchor.pt`.
4. **Overfit to fixed select days.** **FIX:** rotate train/select each generation over a fixed pool;
   permanent audit holdout.
5. **Config one-door / two-inputs.** base values read from configs; focus split single-sourced in
   `auto_ranges`; envelope framing corrected; removed dead config knobs.
6. Correctness nits: mutate CUDA-generator device; probe `sim.w` snapshot/restore; atomic saves;
   `best_streak` init 0 (no trivial `pass0000`); CRN `day_idx` alignment assert.

Verification: **34/34 pytest pass**, gate 5/5, pseudo-replication unit test passes, CPU end-to-end
smoke + resume verified. **These prove the MACHINE is correct — not profitability.**
